## What this is
A **static** site on **GitHub Pages** (repo `PhoenixAB88/pulsar-network`, main/root) at the custom
domain **pulsarnetwork.xyz**. It's a live, real-time monitor of the Xandeum pNode network plus an
Insights article section. "Static site, live data": the page is static HTML; the live numbers come
from small JS data files regenerated by cron on two generator nodes and committed alongside.

## Golden rules
1. **Edit `Pulsar_Network.html`** (the canonical source). Before deploy, `cp Pulsar_Network.html index.html` — GitHub Pages serves `index.html`.
2. **Never hand-edit the cron-generated data files** (they're overwritten every cycle): `network_data*.js`, `network_history*.js`, `stake_data.js`, `net_storage.js`, `node_storage.js`, `credits_data.js`, `node_stats_*.js`.
3. **Never commit secrets**: Helius RPC key, Telegram bot token, etc. They live in PRIVATE files / on the nodes only.
4. **Watch deploy collisions** (see below) — push when the crons aren't mid-cycle.

## Deploy (from a machine with the repo + SSH key `~/.ssh/xandeum`)
```
cp Pulsar_Network.html index.html
rsync -az -e "ssh -i ~/.ssh/xandeum" ./ root@62.171.187.91:/root/pulsar-network/
ssh -i ~/.ssh/xandeum root@62.171.187.91 'cd /root/pulsar-network && git add -A && git -c user.name=Pulsar -c user.email=bot@users.noreply.github.com commit -q -m "deploy" && git pull -q --rebase origin main && git push -q origin main && echo PUSHED'
```
- The node clone needed `git config --global --add safe.directory /root/pulsar-network` once.
- **Deploy collisions:** two cron generators (devnet `62.171.187.91`, mainnet `13.140.143.120`) commit+push every ~10-15 min. A manual push interleaving with a cron push can cancel the Pages build; the site stays on the last green build (new files 404) until one push completes uninterrupted. Push when idle, or just re-run.

## Cron generators (Python; live in `Xandeum Nodes/`, scp'd to the nodes)
- `pulsar_build.py` → `network_data*.js` (get-pods roster, geolocated).
- `pulsar_stake.py` → `stake_data.js` (on-chain stake via Helius, server-side; devnet node).
- `pulsar_history.py` → `network_history*.js` (rolling uptime; state file kept OUTSIDE the repo).
- `pulsar_alerts.py` → Telegram down-node alerts (mainnet node + sibling watchers).
- `pulsar_addself.py` → injects the generator's own node into get-pods (a node omits itself).
- `pulsar_storage_direct.py` → `net_storage.js`/`node_storage.js` (polls each :6000 get-stats; PARTIAL — full network storage needs Atlas, parked).
- `pulsar_credits.py` → `credits_data.js` (OFFICIAL credits from podcredits.xandeum.network; flat pubkey→credits map; powers the Health column).

## Data the frontend reads (window globals)
`NETWORK_*`, `STAKE/STAKE_TOTAL/REG/CUR_YUGA`, `NODE_UPTIME*`, `NODE_HIST*`, `__NS` (net storage),
`CX` (per-node storage), `PODCREDITS` (`.creds`,`.max`), `NODE_STATS`, `XAND_PRICE/SOL_PRICE` (CoinGecko, client-side).

## pRPC + data sources
- pRPC: `POST http://<node-ip>:6000/rpc`, methods `get-pods` / `get-stats` (HYPHENS). Most nodes keep :6000 localhost-only, so direct polling reaches a subset; network-wide storage = Atlas (Foundation, on the asks list).
- Official credits API: `https://podcredits.xandeum.network/api/pods-credits` (public, CORS, no CAPTCHA).
- Do NOT depend on ChillXand (competitor; removed).

## Security
CSP via `<meta>` (GitHub Pages can't set headers; frame-ancestors/HSTS need a proxy). SRI on the
pinned Leaflet CDN. `esc()` + `b58()` sanitize gossip values before innerHTML. Key-only SSH on nodes.

## OG / social cards
`og-<slug>.png` (1200x630) are generated by `Xandeum Nodes/make_og_cards.py` (Python/PIL, premium
code-rendered — not Canva). Edit its `CARDS` list and re-run; deploy the `og-*.png`. **X posting:
text + link only, NO image attachment** (attachments kill the link card); cache-bust the URL (`?v=N`)
to force a fresh crawl.
